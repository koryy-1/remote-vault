# space navy control system
на локальном компе где запущен клиент игры должны работать прослойка и локальный сервак
в локальном сервере будут алгоритмы на случай внештатных ситуаций (напр удаленный сервер не отвечает):
- если заскрамбили и нельзя отварпать, бот должен отбиваться от источника скрамблера
- если в космосе то должен докнуться на станцию

при запуске прослойки сначала должна быть проверка что локальный сервак работает


проекты:

- мб назвать bot-coordinator

- возможно тоже будет управляться ботом верхнего уровня (strategy-bot)
- мб для strategy-bot должно быть без разницы управляет ли он eve-bot-v2.0 или flotilia-control-system

Specs
- подконтрольные боты исполняют команды 
	- уничтожение цели
	- аппроч/орбита цели
	- аппроч/орбита главного бота
	- Алигн/отварп на станцию должен быть на стороне подконтрольных ботов
	- если отварпали значит должны сами приварпать (конечно если боту ничего не угрожает) - должен быть конфиг
- ну и так далее 
	- алигн/варп на спот
	- Джамп в гейт
	- какой то синк между ботами

фронт посылает команду на уничтожение цели
бекенд (SNCS) принимает команду
и ее рассылает всем иви ботам из группы
иви боты исполняют команду
бек ждет когда команда не исполнится, периодично посылая статус выполнения команды ботам
фронт также периодично посылает статус беку

функционал

	- через воркер
	- воркер будет обновлять текущий BotStatus, BotConfig, Command и IsOnline

	- если команда никакая не стоит, значит бот выполнил ее
